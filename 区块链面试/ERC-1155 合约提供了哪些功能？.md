
ERC-1155 是以太坊上一个非常灵活的**多代币标准**，相比 ERC-20、ERC-721，它一次性解决了“多类型代币管理”的痛点。下面我按你作为区块链开发者的角度，给你总结 **ERC-1155 合约核心功能（标准规定的 API）+ 实际开发时你能用到的能力**。

# ✅ **ERC-1155 提供的核心功能（标准 API）**

### **1. 批量转账功能（Batch Transfer）**

这是 1155 最大的优势。

- `safeTransferFrom(address from, address to, uint256 id, uint256 amount, bytes data)`
    
- `safeBatchTransferFrom(address from, address to, uint256[] ids, uint256[] amounts, bytes data)`
    

可一次性转移多个代币类型或多个数量，节约 Gas。

# ✅ **2. 批量查询（Batch Balance Query）**

- `balanceOf(address account, uint256 id)`
    
- `balanceOfBatch(address[] accounts, uint256[] ids)`
    

一次查多个地址 + 多个代币 ID 的余额。

# ✅ **3. 多类型代币统一管理**

ERC-1155 可以同时创建：

- **同质化代币（Fungible Token）**：例如游戏里的金币（ID=1）。
    
- **半同质化代币（SFT）**：例如同一装备的多个副本（ID=2）。
    
- **非同质化代币（NFT）**：ID 唯一的稀有物品（ID=10001）。
    

全部用一个合约实现，而不是多个 ERC-721 / ERC-20 合约。

# ✅ **4. 铸造（Mint）与批量铸造（Batch Mint）**

OpenZeppelin 实现：

- `_mint(address to, uint256 id, uint256 amount, bytes data)`
    
- `_mintBatch(address to, uint256[] ids, uint256[] amounts, bytes data)`
    

可以一次性生成大量代币，适合游戏道具、箱子批量发放等场景。

# ✅ **5. 销毁（Burn）与批量销毁（Batch Burn）**

- `_burn(address from, uint256 id, uint256 amount)`
    
- `_burnBatch(address from, uint256[] ids, uint256[] amounts)`
    

适合需要回收资源/销毁物品的游戏类项目。

# ✅ **6. 安全传输机制（Safe Transfer）**

传输时会检查：

- 接收方如果是智能合约 → 必须实现 `IERC1155Receiver`  
    否则交易 revert，避免代币丢失。
# ✅ **7. 批准（Approval）机制**

统一授权所有 token IDs 的操作权限：

- `setApprovalForAll(address operator, bool approved)`
    
- `isApprovedForAll(address account, a90ddress operator)`
    

和 ERC-721 一样，但更高效。

# ✅ **8. 元数据 URI 管理（Metadata URI）**

- `uri(uint256 id)`
    

1155 的 URI 一般包含占位符：

`https://api.project.com/metadata/{id}.json`

钱包会自动替换 `{id}`。

支持不同 ID 拥有不同 metadata。

# ⚙️ 附加功能（OpenZeppelin 扩展）

在实际开发中经常用到：

### **9. Pausable（可暂停合约）**

暂停所有转账，防止攻击扩散。

### **10. Supply Tracking（跟踪总量）**

扩展包里有：

- `ERC1155Supply`
    

实现：

- `totalSupply(id)`
    
- `exists(id)`
    

如果你做 NFT、道具，就非常有用。

### **11. AccessControl（角色权限）**

如 MINTER_ROLE、BURNER_ROLE，实现多管理员系统。

# 📌 ERC-1155 适用于哪些应用？

由于多代币合一，它非常适用：

- GameFi 游戏道具系统（常用）
    
- NFT + SFT 混合系列
    
- 盲盒、装备、积分、材料类型等多 token 模型
    
- 市场类应用（拍卖行、道具交易）
# 🔥 总结一句话

> **ERC-1155 = 一个合约同时管理 ERC-20 + 多个 ERC-721 + 多个 SFT，并支持批量操作 + 更省 Gas。**
