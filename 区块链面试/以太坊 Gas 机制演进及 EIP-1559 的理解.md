# ✅ **EIP-1159 对 Gas 的变化**

### 1️⃣ **传统 Gas 模型（前 EIP-1559）**

- 用户发送交易时，需要自己指定：
    
    1. **Gas Limit**：最多消耗多少 Gas
        
    2. **Gas Price**：每单位 Gas 的价格（Gwei）
        
- **问题**：
    
    - Gas price 不透明 → 用户常付过高手续费或交易延迟
        
    - 网络拥堵时，交易费高且不稳定
        
    - 矿工收入完全依赖 Gas price → 有投机性
### 2️⃣ **EIP-1559 核心变化**

1. **基本费 Base Fee**
    
    - 区块协议自动计算一个 **基础 Gas 费**
        
    - 随网络拥堵动态调整
        
    - **销毁（Burn）** → 减少 ETH 供应量
        
2. **小费 Tip / Priority Fee**
    
    - 用户可额外支付小费，激励矿工优先打包交易
        
3. **最大费用 Max Fee**
    
    - 用户设置交易的最大愿意支付金额
        
4. **Gas 计算公式**
    
    `Total Fee = Base Fee + Tip`
    
    - Base Fee → 系统自动消耗并销毁
        
    - Tip → 矿工获得
        
5. **动态区块大小**
    
    - 区块大小可在 0–2 倍目标区块 Gas limit 之间浮动
        
    - 高峰区块可临时增大 Gas 承载
### 3️⃣ **EIP-1559 优势**

- **用户体验提升**
    
    - 不必频繁查询 Gas Price
        
    - 交易费用更可预测
        
- **ETH 通缩效应**
    
    - Base Fee 被销毁 → ETH 总量减少
        
- **减轻矿工投机**
    
    - 通过 Base Fee 自动调节网络拥堵
        
- **支持 Layer2 / Rollup**
    
    - 提供更稳定 Gas 模型，方便二层结算
        

---

### 4️⃣ **高分总结回答模板**

> 在 EIP-1559 之前，以太坊交易费完全由用户自己设置 Gas Price，导致交易费用不透明且波动大。  
> EIP-1559 引入了 **Base Fee（基础费用） + Tip（优先费）** 机制：Base Fee 由协议动态计算并销毁，Tip 给矿工激励，用户只需设置 Max Fee。  
> 这样不仅提升了交易费用可预测性，还带来 ETH 通缩效应，同时简化了用户体验和支持 Layer2 扩展。
