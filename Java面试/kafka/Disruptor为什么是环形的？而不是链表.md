### 🔥 **1. 固定内存、无 GC、无扩容**

RingBuffer 预先分配一块连续内存，写满后从头覆写。

- **没有动态扩容**
    
- **没有对象频繁创建回收**
    
- **避免 GC 抖动**
    

> 这对高频交易系统、低延迟系统尤其重要。

### 🔥 **2. 利用 CPU Cache 线性访问 → 极致性能**

环形结构的数组在内存中是连续存储，CPU 更容易预测访问下一条数据，提高缓存命中。

相比队列链表结构，访问下一节点可能跳地址，cache miss 多、延迟大。

---

### 🔥 **3. 无锁并发靠序号定位元素**

Disruptor 不是靠 next 指针，而是靠 **Sequence 序号 % ringBufferSize** 来定位槽位：

`index = sequence & (bufferSize - 1)   // bufferSize必须是2^n`

这带来：

|特性|带来的好处|
|---|---|
|通过序号算索引而非 next 指针|**极低寻址成本**|
|多生产者、多消费者无锁并发|**避免锁竞争，延迟可控**|
|可直接覆盖过旧数据（若处理完）|**无限流式持续运行**|

---

### 🔥 **4. 真正的原因不是“防止溢出”，而是**：

✔ **空间固定可复用**  
✔ **连续内存适配 CPU Cache**  
✔ **基于序号无锁并发访问**  
❗是否覆盖旧数据由 `Sequence` + `GatingSequence` 判断，不是靠结构防止溢出。
