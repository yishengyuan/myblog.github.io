在区块链领域中，「白皮书（Whitepaper）」与「黄皮书（Yellowpaper）」都是重要的技术文档，但定位、内容深度、读者对象完全不同。可以简单理解为：

🔹**白皮书 = 产品说明书**  
🔹**黄皮书 = 技术规格 + 数学证明 + 协议底层原理**

## 🏳 白皮书（Whitepaper）

| 白皮书特点 | 说明                        |
| ----- | ------------------------- |
| 面向对象  | 投资者、社区、开发者、潜在用户           |
| 内容风格  | 偏介绍、概念化、可读性高              |
| 信息重点  | 产品愿景、市场痛点、解决方案、代币经济模型、路线图 |
| 技术深度  | 一般为架构级，高层，不深入公式和底层细节      |
| 常见场景  | 项目融资、展示产品理念、推广和对外宣传       |

📌 举例

- 比特币白皮书：介绍点对点电子现金系统的理念和基本机制
    
- 以太坊白皮书：讲区块链不仅可转账，还能跑智能合约
    

白皮书通常是让别人**看懂这是个什么项目、为什么有价值、未来如何**。

## 🟡 黄皮书（Yellowpaper）

| 黄皮书特点 | 说明                                |
| ----- | --------------------------------- |
| 面向对象  | 核心开发者、研究人员、协议设计人员                 |
| 内容风格  | 学术化、数学严谨、参数公式密集                   |
| 信息重点  | 底层共识算法、虚拟机结构、状态转换过程、数据格式、opcode规范 |
| 技术深度  | 极深，覆盖可验证、可实现的协议细节                 |
| 常见场景  | 协议实现、兼容性开发、研究改进、学术分析              |

📌 举例

- 以太坊黄皮书：EVM执行模型、Gas计算、状态树变更的公式化表达
    
- 门罗、ZK项目等黄皮书会包含大量密码学证明
    

黄皮书属于**能照着写出一个完整兼容客户端**的级别，是协议的**标准与数学定义**。

## 一句话总结

|文档|作用|类比|
|---|---|---|
|白皮书|描述产品与愿景，让人理解价值与用途|宣传册 + 产品说明书|
|黄皮书|描述技术规格与数学模型，可用于协议实现|ISO标准/开发手册|

白皮书告诉你「为什么做、要做什么」。  
黄皮书告诉你「底层是怎么实现的，公式和代码怎么写」。

### **📌 1. 交易对结构**

| 术语        | 全称/含义               | 通俗解释                                 |
| --------- | ------------------- | ------------------------------------ |
| **Base**  | Base Currency（基础币）  | 买卖的目标币（你想买/卖的币），如 BTC/USDT 中 **BTC** |
| **Quote** | Quote Currency（计价币） | 用来计价和结                               |


一个交易对由 **Base币 + Quote币** 组成，base是交易资产，quote是计价币。  
交易有价格精度(**pricePrecision**)、数量精度(**amountPrecision**)、最小价格跳动(**tickSize**)与精度scale(**priceScale**)约束。  
盘口分为 **bid买单**和**ask卖单**，maker下挂单产生深度，taker吃单撮合成交。  
行情数据分为 **depth(盘口深度)、ticker(行情摘要)、kline(K线)、trade(逐笔成交)**。
